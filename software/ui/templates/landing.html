Core Features

Architecture Overview

🔐 Per-Node Identity

Each node holds a unique public/private key pair. The public key identifies the node; the private key signs every outgoing message for tamper-proof authenticity.

📡 Smart Mesh Routing

Nodes score neighbors by GPS proximity, LoRa signal strength, and last-seen time. Periodic beacons propagate presence and coordinates across the mesh automatically.

🛡️ Hop-by-Hop Verification

At every relay hop, the forwarding node verifies the previous sender is an authorized network member before accepting and forwarding the message.

🔄 Duplicate Suppression

Messages carry a (sender_id, seq_num) pair. Nodes track seen IDs and silently drop already-processed packets to prevent loops.

📍 Friendly Force Tracking

All friendly vessels, drones, and stations maintain a live, authenticated position picture of every other node — no central server needed.

⚡ Packed C Structs

Messages serialize as packed C structs to minimize over-the-air byte count — critical for LoRa's limited bandwidth and packet size constraints.

Verification Layer

Active

Every node is a cryptographic identity. Public keys are pre-programmed into every other node's memory before deployment — no key exchange needed in the field.

✦  Unique keypair per node ✦  Public keys pre-shared before deployment ✦  Private key signs every outbound message ✦  Fallback: shared network key to reduce packet size

Routing Layer

Active

The network is self-organizing. Each node maintains a scored neighbor table updated continuously by beacons, RSSI readings, and GPS coordinates.

✦  Periodic beacons broadcast position + presence ✦  Neighbors scored by proximity, RSSI, recency ✦  Best-path algorithm selects optimal next hop ✦  No single point of failure
